## Diagnóstico (situação atual)

- 14 tabelas em `public` com RLS ativo
- 13 delas (todas exceto `tenants`) têm policy `phase1_anon_all` com `USING (true) WITH CHECK (true)` — qualquer um com a anon key faz qualquer coisa via REST direto
- `tenants` tem policy `phase1_anon_select_tenants` (só SELECT) — correto, precisa ler antes da sessão
- `tenant_users` tem RLS ativo mas zero policies — INFO no advisor (sem linhas, então tanto faz por ora)
- Função `is_member_of_tenant(uuid)`:
  - `SECURITY DEFINER` sem `search_path` setado → WARN `function_search_path_mutable` (risco baixo, mas best practice setar)
  - Executável pelo role `anon` via `/rest/v1/rpc/is_member_of_tenant` → WARN `anon_security_definer_function_executable`
- Front (`src/lib/supabase.ts`) usa `persistSession: true, autoRefreshToken: true` — Supabase JS troca o role na request de `anon` pra `authenticated` automaticamente assim que houver JWT válido, sem mudar uma linha de código

## Rollback

Se algo der errado, rodar:

```sql
drop policy if exists "authenticated_all" on public.<tabela>;
-- (para cada tabela)
drop policy if exists "authenticated_select_tenants" on public.tenants;
-- A função volta ao estado original com:
alter function public.is_member_of_tenant(uuid) reset search_path;
grant execute on function public.is_member_of_tenant(uuid) to anon, public;
```
