<objective>
## Goal
Entregar a primeira seção de **análise cruzada** do dashboard: um painel que correlaciona **investimento Meta Ads** e **tráfego (cliques)** com **faturamento diário de vendas**, no período selecionado nos filtros globais. Conecta diretamente com o objetivo central do projeto ("cruzamento de dados entre fontes para identificar tendências e correlações") e com a métrica de sucesso "Correlações entre métricas visíveis: pelo menos 2 cruzamentos". Entrega valor real pro Doug e pro time de assessoria: "quando eu invisto mais, vende mais?" deixa de ser palpite e vira gráfico + número.

## AC-1: Função pura de correlação de Pearson

```gherkin
Given duas séries numéricas x e y de mesmo tamanho
When chamo pearson(x, y)
Then retorna o coeficiente de Pearson (número entre -1 e 1)
  And se as séries tiverem menos de 2 pontos válidos, retorna null
  And ignora pares onde x ou y é null/NaN
  And tem teste mental simples: pearson([1,2,3], [2,4,6]) === 1 (correlação perfeita)
```

## AC-2: Interpretação textual do coeficiente

```gherkin
Given um coeficiente r (ou null)
When chamo interpretCorrelation(r)
Then retorna { label, color, direction } com a classificação:
  - |r| < 0.1 → "Sem correlação aparente" (cinza)
  - 0.1 ≤ |r| < 0.3 → "Correlação fraca" (cinza-azul)
  - 0.3 ≤ |r| < 0.5 → "Correlação moderada" (azul)
  - 0.5 ≤ |r| < 0.7 → "Correlação forte" (verde)
  - |r| ≥ 0.7 → "Correlação muito forte" (verde-escuro)
  - direction = "positiva" se r > 0, "negativa" se r < 0
  - r === null → { label: "Dados insuficientes", color: cinza, direction: null }
```

  <action>
    Criar arquivo novo com 2 exports:

    1. `pearson(xs: number[], ys: number[]): number | null`
       - Valida tamanhos iguais; se diferentes, lança erro de programação
       - Filtra pares onde x ou y é null/NaN/undefined ANTES de calcular
       - Se restarem < 2 pontos, retorna null
       - Calcula via fórmula clássica:
         r = Σ((xi - x̄)(yi - ȳ)) / sqrt(Σ(xi - x̄)² · Σ(yi - ȳ)²)
       - Se denominador é 0 (uma série constante), retorna null (correlação indefinida)
       - Garante retorno entre -1 e 1 (pode ter erro de ponto flutuante; clampa)

    2. `interpretCorrelation(r: number | null): { label: string; color: string; direction: 'positiva' | 'negativa' | null }`
       - Conforme AC-2
       - Cores: use as CSS vars existentes quando possível (--accent-purple, --positive-green, --alert-red, --text-muted, etc.) — se não existirem nomes apropriados, retorne hex direto e adiciono var no CSS no Task 3
