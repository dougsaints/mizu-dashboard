<objective>
## Goal
Portar a **"Tabela de Dados"** do painel HTML original (linhas 2107-2137) pro app React. Duas partes:
1. **Matriz Mês-a-Mês (MoM)** — comparação fixa "mês atual × mês anterior" por unidade, com Δ% por canal (PDV/iFood/AnotaAi), independente do filtro de período global. Lógica em `renderMomMatrix()` (HTML linha 4690-4773).
2. **Tabela agregada** — linhas com Período · Unidade · PDV · iFood · AnotaAi · Total · Dias, baseada no filtro global. Botão "⬇ Exportar CSV" que baixa o conteúdo da tabela.

## Purpose
- O painel HTML tem essa visualização tabular que falta no app — gestores adoram tabela pra colar em reunião ou exportar.
- A matriz MoM dá um pulse "como esse mês está vs o anterior" sem precisar mexer no filtro de período.
- Export CSV abre caminho pra outros exports futuros (já cria a infraestrutura `csvExport.ts`).

## Output
- `src/lib/aggregation.ts` — funções puras de agregação por unidade/canal + helpers de range mensal
- `src/lib/csvExport.ts` — gerador de CSV (string + download via Blob)
- `src/sections/DataTableSection.tsx` — seção com matriz MoM + tabela agregada + botão export
- Wire-up no `Dashboard.tsx` (após `CorrelationSection`, antes de `RoiSection`)
- CSS apendado no `index.css`
- `.paul/phases/06-correlacoes-analise-cruzada/06-03-SUMMARY.md` ao final

  <name>Task 1: Criar lib/aggregation.ts — helpers de agregação mensal</name>
  <files>src/lib/aggregation.ts</files>
  <action>
    Funções puras (sem React, sem Supabase):

    ```ts
    type SalesRow = { unit_id: string; date: string; pdv: number; anotaai: number; ifood: number; total: number }
    type MonthRange = { start: string; end: string }
    type UnitAgg = { total: number; pdv: number; ifood: number; anotaai: number; count: number }

    export function currentMonthRange(today: Date): MonthRange
    export function prevMonthRange(today: Date): MonthRange
    export function aggregateUnitMonth(rows: SalesRow[], range: MonthRange): UnitAgg
    export function computeDelta(curr: number, prev: number, prevDays: number, currDays?: number): {
      pct: number | null      // null se não computável
      direction: 'up' | 'down' | 'flat'
      // Note: NÃO normalizar por dias aqui — mês em curso normalmente terá Δ negativo, e isso é OK
      // (a nota visual no UI explica). Manter a função simples.
    }
    ```

    Detalhes:
    - `currentMonthRange(today)`: start = `yyyy-MM-01`; end = ISO de `today` (não fim do mês)
    - `prevMonthRange(today)`: start = `yyyy-(MM-1)-01`; end = último dia do mês anterior
    - `aggregateUnitMonth`: filtra rows com `date >= range.start && date <= range.end`, soma campos numéricos, count = nº de dias únicos com dados
    - `computeDelta`: if prev === 0 → pct=null (badge "novo" se curr>0). Senão, pct = (curr-prev)/prev * 100. flat se |pct| < 0.5, up se >0, down se <0
    - Lidar com fuso: usar Date com strings ISO sempre que possível pra evitar drift. Helper interno: `addMonths(date, n)`, `lastDayOfMonth(year, month)`

    Avoid:
    - Mutate rows
    - Usar `moment`/`dayjs` (sem nova dep — date-fns ou nativo)
    - Importar de useSales aqui (tem que ser puro)
  </action>
  <verify>`npx tsc --noEmit` passa. Testes mentais:
    - currentMonthRange(new Date('2026-05-23')) → { start: '2026-05-01', end: '2026-05-23' }
    - prevMonthRange(new Date('2026-05-23')) → { start: '2026-04-01', end: '2026-04-30' }
    - aggregateUnitMonth com 3 rows → soma correta + count = 3
    - computeDelta(120, 100, 30) → { pct: 20, direction: 'up' }
  </verify>
