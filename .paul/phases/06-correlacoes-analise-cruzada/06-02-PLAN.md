  <action>
    Criar componente puro em `src/components/TrendCard.tsx`:

    ```ts
    type Favorability = 'positive' | 'negative' | 'neutral'
    type TrendCardProps = {
      label: string
      formatted: string       // valor formatado ex: "R$ 12.345,67"
      curr: number | null
      prev: number | null     // null = sem comparação
      formattedPrev?: string  // pra tooltip
      favorability?: Favorability
      hint?: string
    }
    ```

    Lógica do badge (interna):
    - Se `prev === null || prev === undefined` → não mostra badge
    - Se `curr === null` → não mostra badge
    - Se `prev === 0`:
      - Se `curr > 0` → badge "novo" cinza
      - Se `curr === 0` → não mostra badge
    - Senão, `delta = (curr - prev) / prev * 100`
      - `|delta| < 0.5` → flat (cinza)
      - `delta > 0`: ▲ — se favorability='positive' verde; 'negative' vermelho; 'neutral' azul
      - `delta < 0`: ▼ — se favorability='positive' vermelho; 'negative' verde; 'neutral' azul

    Layout:
    - Card pequeno (`.trend-card`) com:
      - `.trend-card-label` (pequeno, cinza)
      - `.trend-card-value` (grande, bold)
      - badge ao lado do valor ou abaixo
      - `.trend-card-hint` opcional (cinza)

    Avoid: depender de cmpMode aqui — quem decide se compara é o pai (TrendsSection). Esse componente só recebe curr/prev.
  </action>
