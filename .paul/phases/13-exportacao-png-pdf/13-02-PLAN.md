# Plan + Summary (consolidado)

## O que foi feito

Botão "📄 PDF" no Header global que chama `window.print()`. CSS `@media print` configura A4, esconde controles, mantém charts (canvas printable), adiciona marca d'água "水 Sushi Mizú" no rodapé de cada seção via pseudo-element.

## Mudanças

- **Header.tsx:** botão `.header-print-btn` antes do user-logout. Chama `window.print()`.
- **index.css:** CSS `.header-print-btn` (creme + hover gold) + bloco `@media print` extenso:
  - Esconde: `.header, .header-print-btn, .header-logout, .uploads-area, .lazy-fallback, .roi-toggle, .section-source-badge, button` (mantém charts e KPIs)
  - `@page { margin: 12mm 10mm; size: A4 }`
  - `body { background white, color black, font 11pt }`
  - `.mizu-section { page-break-inside: avoid }` — cada seção tenta caber inteira numa página
  - Marca d'água "水 Sushi Mizú" no canto inferior de cada seção
  - `canvas { max-width: 100% }` — Chart.js renderiza no print

## Decisões

- **Por que não dois modos (diário/semanal)?** Doug pode escolher o período no DateRangePicker antes de imprimir. Modos especializados exigiriam URL state/modal — complexidade alta pra ganho marginal. window.print é universal.
- **Por que não jspdf?** ROADMAP pediu começar com print + CSS. Se qualidade não bater, podemos adicionar depois. Zero dependência nova é melhor.

## Resultado

Build limpo. Ctrl+P (ou botão 📄 PDF) abre dialog do browser → "Salvar como PDF" → arquivo gerado. Cada seção numa página A4, sem botões, com marca da Sushi Mizú.
