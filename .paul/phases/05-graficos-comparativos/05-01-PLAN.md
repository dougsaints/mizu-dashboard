<objective>
## Goal
Adicionar um gráfico de linha de faturamento diário na seção Vendas, com
uma linha por unidade (Serraria e Jatiúca), respeitando o seletor de
período já existente (7/30/60 dias).

## Purpose
A seção Vendas hoje só mostra números agregados (KPIs e cards por
unidade). Um gráfico de linha deixa a tendência visível — Doug e o time
veem de relance se as lojas estão subindo ou caindo, e comparam o
desempenho das duas lado a lado. É o primeiro passo da Fase 5
(visualização e comparativos).

## Output
- Novo componente `src/components/SalesLineChart.tsx` (gráfico Chart.js)
- `SalesSection.tsx` renderiza o gráfico abaixo dos cards
- CSS do container do gráfico appendado ao `index.css`
</objective>

## Notas técnicas
- `chart.js@4.5` e `react-chartjs-2@5.3` JÁ estão instalados no
  `package.json` — não instalar nada, só importar e usar.
- Chart.js v4 é "tree-shakable": é preciso registrar manualmente os
  módulos usados (`CategoryScale`, `LinearScale`, `PointElement`,
  `LineElement`, `Tooltip`, `Legend`) antes de renderizar.
- `sales_daily` tem os campos: `date` (ISO yyyy-mm-dd), `unit_id`,
  `total`, `pdv`, `anotaai`, `ifood`.
- `useSales(daysBack)` retorna as linhas ordenadas por data DESC.
- Formato BR: valores em "R$ 1.234,56" (`toLocaleString('pt-BR', …)`),
  datas no eixo X como `dd/mm`.

## AC-1: Gráfico de linha por unidade
```gherkin
Given a seção Vendas com dados de faturamento de Serraria e Jatiúca
When a página carrega
Then aparece um gráfico de linha com o eixo X em datas (dd/mm) e uma

     Serraria e Jatiúca
```

## AC-2: Gráfico respeita o seletor de período
```gherkin
Given o gráfico de vendas renderizado
When o usuário clica em "7 dias", "30 dias" ou "60 dias"
Then o gráfico se redesenha mostrando apenas o intervalo escolhido,
     em sincronia com os KPIs e cards já existentes
```

## AC-3: Tooltip e valores em formato BR
```gherkin
Given o gráfico renderizado
When o usuário passa o mouse sobre um ponto
Then a tooltip mostra a data e o faturamento daquele dia formatado
     como "R$ 1.234,56", e o eixo Y também usa separador BR
```

## AC-4: Estados de carregando / vazio
```gherkin
Given a seção Vendas sem dados ou ainda carregando
When a seção é exibida
Then o gráfico não quebra a página — ele simplesmente não aparece
     enquanto não há linhas (os avisos de "carregando" e "vazio" já
     existentes continuam valendo)
```

  <action>
    Criar um componente de gráfico de linha reutilizável.
    - No topo do arquivo, registrar os módulos do Chart.js v4 com
      `ChartJS.register(CategoryScale, LinearScale, PointElement,
      LineElement, Tooltip, Legend)` — fazer isso uma vez no escopo do
      módulo, não dentro do componente.
    - Props: `rows: SalesRow[]` e `units` (lista de unidades, pra pegar
      `display_name` por `unit_id`). Tipar via `Database['public']
      ['Tables']['sales_daily']['Row']`, sem `any`.
    - Construir os dados do gráfico:
      · Eixo X = todas as datas distintas presentes em `rows`, ordenadas
        ASCENDENTE (as rows vêm DESC do hook — reordenar). Rótulo `dd/mm`.
      · Um dataset por `unit_id`: para cada data, o valor é o `total`
        daquela unidade naquele dia (0 ou null se não houver linha).
      · Cor distinta por unidade — usar 2 cores fixas legíveis (ex.:
        uma quente e uma fria) já que há 2 unidades; se aparecer uma 3ª,
        cair num array de cores.
      · `label` do dataset = `display_name` da unidade.
    - Configurar `options`: tooltip e ticks do eixo Y formatando número
      como BRL via `toLocaleString('pt-BR', { style:'currency',
      currency:'BRL' })`; `responsive: true`,
      `maintainAspectRatio: false`.
    - Renderizar `<Line data={...} options={...} />` dentro de uma div
      com `className="sales-chart-canvas"`.
    - Se `rows` estiver vazio, retornar `null` (AC-4).
    Evitar: registrar módulos dentro do corpo do componente (re-registra
    a cada render); usar `any`; assumir que toda data tem linha das duas
    unidades.
