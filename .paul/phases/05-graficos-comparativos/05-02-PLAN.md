<objective>
## Goal
Dar visualização gráfica ao Meta Ads e ao delivery (Anota AI), que hoje
só têm a caixa de upload. Meta Ads ganha dois gráficos (gasto por dia e
impressões/alcance) com seletor de período; o delivery ganha um gráfico
de barras dos produtos mais vendidos.

## Purpose
Fecha a Fase 5: depois de vendas (05-01), as duas fontes restantes —
tráfego pago e delivery — passam a ter leitura visual rápida. Doug e o
time veem o ritmo do investimento em anúncio, o alcance das campanhas e
quais itens mais saem no delivery, sem ler tabelas.

## Output
- `src/components/AdsCharts.tsx` — gráficos do Meta Ads (gasto + alcance)
- `src/components/TopProductsChart.tsx` — barras de produtos mais vendidos
- `AdsUploadCard` e `AnotaaiUploadCard` renderizam os gráficos
- CSS dos containers appendado ao `index.css`
</objective>

## Notas técnicas
- `chart.js` + `react-chartjs-2` já instalados. Gráfico de barras
  precisa registrar `BarElement` (além dos módulos do 05-01).
- `ads_daily` tem VÁRIAS linhas por data (uma por campanha/ad_set).
  Campos: `date`, `cost`, `impressions`, `clicks`, `reach`, `results`,
  `result_value`. Para os gráficos, agregar (somar) por `date`.
- `useAds(daysBack)` retorna linhas ordenadas por data DESC.
- Top produtos: usar SÓ o snapshot mais recente — é o que o
  `useWeeklyRecap` faz (`latestSnapshot = products[0]?.snapshot_date`,
  filtrar por essa data, somar `quantity` por `product_name`). Cada CSV
  do Anota AI é um relatório acumulado; somar várias fotos conta dobrado.
- `useAnotaaiProducts()` já retorna ordenado por `snapshot_date` DESC.
- Seletor de período: reaproveitar a classe CSS `.range-picker` já
  usada na `SalesSection` (7/30/60 dias) — não criar CSS novo pra ele.
- Padrão de gráfico: copiar a estrutura de `SalesLineChart.tsx`
  (registro de módulos, `useMemo`, `maintainAspectRatio:false`,
  retorna `null` sem dados, formatação BR).
